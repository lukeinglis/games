# Experiment #2: Migrate 5 mini-games into portal with localStorage leaderboards

## Result
**KEEP** — capability_surface jumped from 0.2 to 0.75. 5 games successfully migrated and functional. 2 QA iterations (initial: 6 lint errors, Builder fixed them, final pass clean). PR #4 open for human review.

## What Changed
- 5 game components migrated from 3 source repos (WorldCupFantasy, FantasyFootball, F1 Fantasy)
- All auth/API dependencies stripped (Yahoo OAuth, NextAuth, anti-cheat tokens)
- GameLeaderboard rewritten from API-based to localStorage-based persistence
- 5 new route pages with shared GamePageWrapper layout
- games.ts expanded to 8 total games across 5 categories (Sports, Racing, Trivia, Prediction, Fantasy)
- GameCard updated with Next.js Link for internal routes
- 48 teams data inlined in GuessTheFlag component (removed source repo dependency)

## Build & QA
- Build: PASS (9/9 routes compile)
- Lint: 0 errors (after fix iteration), 3 warnings
- E2E: All routes functional, external links preserved
- QA iterations: 2 (Builder resolved 6 initial lint errors, final pass clean)

## What We Learned
localStorage-only leaderboard eliminates infrastructure cost for friends-league games; inlining static data (teams, configs) removes source-repo coupling and reduces cold-start complexity. Game logic stays 100% intact when dependencies are stripped at the boundaries.

## Deferred Items
- Game thumbnail images (currently emoji placeholders)
- Homepage high-score display from localStorage scores
